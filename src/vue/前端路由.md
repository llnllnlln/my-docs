### SPA是什么
1. SPA(single page web application),单页Web应用,简单来说就是一个web项目只有一个HTML页面，一旦页面加载完成，就不会因为用户的操作而进行页面的重新加载或跳转，而是利用 JS 动态的变换 HTML 的内容来模拟多个视图间跳转。所以SPA允许在不刷新页面的情况下更改URL并渲染不同内容。
2. 旧的网站是每个HTML文件都是一个完成的HTML页面，而在SPA 的应用设计中，一个应用只有一个HTML文件。
3. SPA 的出现大大提高了 WEB 应用的交互体验，但是SPA是通过 JS 改变 HTML 内容来实现的，页面本身的 url 并没有变化，这样会导致刷新、前进还是后退无法被浏览器记录；并且只有一个 url，对 SEO 不友好。
### 为什么要有前端路由？
1. SPA在只有一个HTML页面下，通过 JS 改变 HTML 内容来实现的，页面的url不会变化的，这时候为SPA的每个视图匹配一个特殊的url，那么刷新、前进、后退和SEO就可以通过这个特殊的 url 来实现。
2. 这时候改变url不会让浏览器向服务器发送请求，并能监听url的变化，这种有hash 模式和 history 模式。
### hash模式
1. hash是什么：是在url里面的#以及后面的字符`http://example.com/#/about`
2. 特点：改变url不会让浏览器向服务器发送请求,并且`hash`改变会触发`hashchange`事件，`window.onhashchange`可以监听到浏览器的前进后退
3. hash的获取与设置，以及监听
```js
window.location.hash = 'hash字符串'; // 用于设置 hash 值
let hash = window.location.hash; // 获取当前 hash 值
// 监听hash变化，点击浏览器的前进后退会触发
window.addEventListener('hashchange', function(event){ 
    let newURL = event.newURL; // hash 改变后的新 url
    let oldURL = event.oldURL; // hash 改变前的旧 url
},false)
```
4. 实例
```js
// 定义 Router  
class Router {  
  constructor () {  
    this.routes = {}; // 存放路由path及callback  
    this.currentUrl = '';  
    // 监听路由change调用相对应的路由回调  
    window.addEventListener('load', this.refresh, false);  
    window.addEventListener('hashchange', this.refresh, false);  
  }  
  route(path, callback){  
    this.routes[path] = callback;  
  }  
  push(path) {  
    this.routes[path] && this.routes[path]()  
  }  
}  
// 使用 router  
window.miniRouter = new Router();  
miniRouter.route('/', () => console.log('page1'))  
miniRouter.route('/page2', () => console.log('page2'))  

miniRouter.push('/') // page1  
miniRouter.push('/page2') // page2
```
### history模式
1. 原理：使用HTML5 History API
2. 特点：URL更简洁：`http://example.com/about`；需要服务器端支持(避免404)；不支持IE9及以下；可以改变 url 同时，不会刷新页面。
3. 早期histroy的api：后退一页`history.go(-1);` 前进两页`history.go(2);` 前进一页`history.forward();`后退一页`history.back();`
4. HTML5 的规范中新增api：
    - 返回当前状态对象`history.state` 
    - 添加新的状态到历史状态栈`history.pushState()`; 用新的状态代替当前状态`history.replaceState()`。这两个它们分别可以添加和修改历史记录条目。
    - 接收三个参数`（state, title, url`）
    - `state`：合法的Javascript对象，可以用在`popstate`事件中; `url`：任意有效的URL用于更新浏览器的地址栏。
5. 监听
    - 当hash变化时，可以通过`hashchange`进行监听，history变化则无法监听。
    - 可以通过以下知道路径变化：点击浏览器的前进或后退按钮`window.onpopstate`；
    - 点击a标签；在JS代码中触发`history.pushState、history.replaceState`
6. 注意
    - history 在修改url后，虽然页面并不会刷新。
    - 但我们在手动刷新，或通过url直接进入应用的时候，服务端是无法识别这个url的。因为只有一个html文件，服务端在处理其他路径的url的时候，就会出现404的情况。
    - 所以需要在服务端增加一个覆盖所有情况的候选资源：如果URL匹配不到任何静态资源时候服务器配置重定向到index.html
```js
// 定义 Router  
class Router {  
  constructor () {  
    this.routes = {};  
    this.listerPopState()  
  }  
  init(path) {  
    history.replaceState({path: path}, null, path);  
    this.routes[path] && this.routes[path]();  
  }  
  route(path, callback){  
    this.routes[path] = callback;  
  }  
  push(path) {  
    history.pushState({path: path}, null, path);  
    this.routes[path] && this.routes[path]();
```

### 两者模式比较
1. hash 模式相比于 history 模式的缺点：
    - 看起来更丑，相同 hash 值不会触发动作将记录加入到历史栈中，而 pushState 则可以
2. hash 模式相比于 history 模式的优点：
    - 兼容性更好，可以兼容到IE8，无需服务端配合处理非单页的url地址
