### package.json的script中dev运行流程
1. `npm run bootstrap`：`"bootstrap": "yarn || npm i"`, 下载相关的依赖
2. `npm run build:file`：
   - `"build:file": "node build/bin/iconInit.js & node build/bin/build-entry.js & node build/bin/i18n.js & node build/bin/version.js",`
   - 生成一些文件：入口文件 图标 多语言 版本记录等
3. `cross-env NODE_ENV=development`：
   - `cross-env`传递环境变量的库，设置当前环境变量为开发模式
4. `webpack-dev-server --config build/webpack.demo.js`：
   - 指定webpack的配置文件为`build/webpack.demo.js`，并启动
   - 找到入口文件为`./examples/entry.js`，目的是启动一个集成Element的系统
5. `node build/bin/template.js`：执行模版文件
### package.json的script中dist运行流程？组件库的自动化打包流程？
1. `npm run clean && npm run build:file && npm run lint`：
   - "clean": `"rimraf lib && rimraf packages/*/lib && rimraf test/**/coverage",`
   - 查找`rimraf`，先去`node_module`的bin文件下查找有没有它，有的话则执行脚本，删除`packages/*/lib`文件夹下的内容
   - 生成图标；生成入口文件；生成多语言文件；生成记录版本文件
2. `webpack --config build/webpack.conf.js`
   - 在生成环境下以`src/index.js`为入口文件，将整体项目打包到`/lib`下面，输出文件为`index.js`，并且是`umd`规范，能够支持浏览器加载也能支持npm引入
3. `webpack --config build/webpack.common.js`
   - 在生成环境下以`src/index.js`为入口文件，将整体项目打包到`/lib`下面，输出文件为`element-ui.common.js`，并且是`commonjs2`规范，支持require方式引入
4. `webpack --config build/webpack.component.js`
   - 多入口打包，出口文件也有多个。输出文件为组件名.js 并且是commonjs2规范，支持require方式引入
   - 这个是按需加载的前提，我们按需引入组件时候，需要写上`Vue.component(Button.name, Button);`或写为`Vue.use(Button)`
   - Vue.use(Button)原理是`if(componnent.install){ 执行install函数 }`install其实就是`Vue.component(Button.name, Button)`;
5. `npm run build:utils && npm run build:umd && npm run build:theme`
   - `"build:utils"`: `"cross-env BABEL_ENV=utils babel src --out-dir lib --ignore src/index.js"`
   - `"build:umd"`: `"node build/bin/build-locale.js"`, 多语言支持
   - `"build:theme"`: `"node build/bin/gen-cssfile && gulp build --gulpfile packages/theme-chalk/gulpfile.js && cp-cli packages/theme-chalk/lib lib/theme-chalk"`
        - `node build/bin/gen-cssfile`：根据packages下的theme-chalk文件的各个组件的scss文件相对路径自动化生成一个index.scss，是一个css入口文件
        - `gulp build --gulpfile packages/theme-chalk/gulpfile.js` 1 将scss转成css 2 autoprefixer补上兼容的前缀 3 压缩 4 打包到/lib目录下
        - gulp 是一个前端构建工具，是进行压缩 转码 加工的功能单一、小巧易用的建构工具；
        - 为什么混用gulp，为了按需引入，css与js做分离，gulp目的单纯小巧易用并且单类型转换，没必要用webpack
### 组件库按需加载的原理是什么？
1. 使用`webpack --config build/webpack.component.js`实现多入口打包，使每个组件有自己的输出文件
2. 使用gulp工具转换scss文件，将css文件打包到lib目录下
3. 用户引入需要插件：`babel-plugin-component`，它将用户输入的`import { Button, Select } from 'element-ui'`;转为具体的被打包的js、css 路径文件，这样css和js就能被用户使用了。
   - `babel-plugin-component`做了什么事情：将上面引入转为具体地址，`import Button from 'element-ui/lib/Button.js'`;`import 'element-ui/lib/theme-chalk/Button.css'; `
   - 原理是根据组件名曲lib目录下找同名的js组件、同名的css文件。
### webpack.demo.js文件详解
1. `entry：'./examples/entry.js'` 入口文件
2. `output：path.resolve(process.cwd(), './examples/element-ui/')`, 输出到examples的文件夹下
3. `resolve：extensions: ['.js', '.vue', '.json']`, 说明import引入文件时候，这些后缀可以自动补全；alias是路径别称
4. `devServer`: 开启本地端口服务
5. `module`:
```json
  {test: /\.(vue|jsx?)$/,loader: 'eslint-loader'},
  {test: /\.(jsx?|babel|es6)$/,include: process.cwd(),loader: 'babel-loader'},// 把 ES6 转换成 ES5 
  {test: /\.vue$/,.loader:'vue-loader'},
  {test: /\.(scss|css)$/, use: ['style-loader','css-loader','sass-loader']},
    //从右往左解析:sass-loader：sass转css；css-loader：加载 CSS，⽀持模块化、压缩、⽂件导⼊；style-loader：把 CSS 代码注⼊到 JavaScript 中，通过 DOM 操作去加载 CSS
  {test: /\.md$/,use: [ loader: 'vue-loader', loader: path.resolve(__dirname, './md-loader/index.js')]
```
### md-loader文件详解？与组件的文档实例有关。
1. `markdown-it`解析`markdown`的库，将`markdown`代码编译我们可以使用的抽象语法树。
2. `containers.js`：作用是将`:::demo`转为注释节点，将里面内容放在`<demo-block><!--element-demo>`标签里面
3. `index.js`：拿到经过初步处理后的文件后，匹配拿出`<!--element-demo>`里面的内容，包裹在`<template slot="source"></template>`里面，并经过加工，最终变成一个vue文件，
4. 再有`vue-loader`处理。
5. `<demo-block>`包裹了内容，所以实例界面可以并交互并折叠组件
### 如何自动化创建组件？
1. `Makefile`：管理命令，命令的集合入口，如`dev:npm run dev`: `new:node build/bin/new.js $(filter-out $@,$(MAKECMDGOALS))`
2. 用户输入`new select`,那么就会自动去创建新的组件的相关文件，比如select组件的`index文件，vue文件，css文件`等等会自动创建出来，这样子就不用一个一个去创建相关文件，节省重复事情，避免出问题。
3. 在终端输入`new`，系统就 
    - ①执行了`build/bin/new.js`文件，然后根据传入的项目名字参数，生成`packages`下面的组件文件，
    - ②并且把组件名、相对路径添加到添加到`components.json`，
    - ③sass模板内容添加到`index.scss`添加ts文件； 
    - ④添加组件名到`nav.config.json`导航文件等。
4. 很多脚手架的新建组件、新建文件、自动生成配置文件，都有这种自动化流程。目的是减少手动添加文件、引入文件，做到减少重复无聊的步骤。

1. 把`src/locale/lang`下的多语言文件进行打包，用户可以通过`import lang from 'element-ui/lib/locale/lang/en';locale.use(lang)`来引入
2. 其中原理是利用占位符，不同语言占位符可以替换为不同内容
