                              # 常用文档库

                              1. Docusaurus: React的文档库，小巧 好用 支持国际化
                              2. VuePress: vue的文档库，以内容为中心的静态文档库
                              3. Docsify: 不适合SEO优化
                              4. Nuxt: 为构建应用程序而设计的
                              5. elementui组件库的自定义文档是使用自定义vue-loader转vue文件，vue-loader解析为静态资源

                              # Docsify

                              是一个基于 JavaScript 的动态文档网站生成器，它能让你快速搭建一个优雅的文档站点而无需构建静态 HTML 文件

                              # 完整搭建

                              1. 搭建环境：安装 Node.js，安装 Docsify CLI：npm install -g docsify-cli
                              2. 初始化：创建一个新目录并初始化 Docsify：docsify init ./docs；
                                • index.html - 入口文件 README.md- 文档主页内容，
                                • 修改index.html，设置名字、启用侧边栏、栏目录层级、封面页等
                                • name: '我的文档',loadSidebar: true,subMaxLevel: 3,coverpage: true,search: 'auto'
                              3. 编写文档：主页（README.mdopen in new window）、侧边栏（_sidebar.md）、封面（_coverpage.md）
                              4. 本地预览：docsify -v ；docsify serve docs，再访问 http://localhost:3000 查看效果。
                              5. 添加插件资源：在index.html中添加插件资源
                                • 搜索插件 <script src="//cdn.jsdelivr.net/npm/docsify/lib/plugins/search.min.js"></script>
                                • 数学公式 <link rel="stylesheet" href="//cdn.jsdelivr.net/npm/katex/dist/katex.min.css">
                                • 图片缩放 <script src="//cdn.jsdelivr.net/npm/docsify/lib/plugins/zoom-image.min.js"></script>
                                • 搜索: search.min.js; 图片缩放: zoom-image.min.js; 分页导航: pagination.min.js
                              6. 多语言
                              window.$docsify = {
  locales: {
    '/zh-cn/': {
      lang: 'zh-CN',
      title: '中文文档'
    },
    '/en/': {
      lang: 'en',
      title: 'English Docs'
    }
  }
}

                              # Docusaurus

                              • Docusaurus是一个React常用的文档站框架，支持MD和MDX语法。
                              • MDX语法的主要功能是在MD文档里使用JSX语法，可以在md文件使用React组件

                              # 完整搭建

                              1. npx create-docusaurus@latest my-website classic npx:在全局查找库没有就在当前目录下局部安装，是临时的，用完即删
                              2. 文件结构
                                ⅰ. blog：博客的 Markdown 文件
                                ⅱ. docs：文档的 Markdown 文件
                                ⅲ. src/pages：独立页面
                                ⅳ. docusaurus.config.js：配置的配置文件
                                ⅴ. sidebars.js：侧边栏
                              3. 声明是路由：在pages下创建的js文件都会按照pages的层级结构自动转为页面，__开头的文件则是其他文件的链接

                              # Mermind是流程图的工具库，mdx是支持它的这些语法的，但是docusaurus不支持，该怎么解决？

                              1. docusaurus.config.js的详解，如何在文档库个性化定制文档的语法？
                              2. 第一步：docusaurus.config.js引入remark插件docs:remarkPlugins:[require('./src/theme/Mermaid-plugin')]
                              3. 第二步：自定义Mermaid-plugin插件
                                • mdx已经把md文件转成ast树，visit遍历整个树，然后对ast树{ type: 'code', lang: 'mermaid' }进行加工处理，就是用正则匹配和splice替换为<Mermaid chart={${node.value}}>,转为Mermaid的react组件。
                                • insertImport第一次解析不存在的时候则import引入Mermaid库。
                                • 其实目的就是将mermaid graph TD; A-->B; A-->C; B-->D; C-->D; 转成import Mermaid from 'mermaid' <Mermaid name="diagram">graph TD; A-->B; A-->C; B-->D; C-->D; </Mermaid>
                              4. remark plugins：当mdx将md文档转成ast的过程 ，抓住中间时机，做一些直接对md转换的操作语法、个性化定制。
                              5. rehype plugins：ast转换完成之后，加工检查格式。如格式化插件
