        背景

        1. 经过表格数据录入与计算，需要用图表直观展示。

        3. 复杂一点的计算特别注意精度问题和边界情况,不用原生的JS运算符+ - * /直接计算，在浮点数二进制表示方式，可能会导致计算结果不准确，可读性也低。

        大文件上传

        背景

        1. 大文件是指传送大于100M的文件，容易遇到问题：1.前后端上传请求超时限制，一次性传输大小限制。 2.网络抖动等，失败后需要重新上传。为了避免上传速度很慢，传输中断等问题，需要使用切片上传的方式。
        2. 课件支持上传音频、视频、PPT、PDF等类型的课件；上传的音视频课件大于100M的文件，容易遇到上传速度很慢，传输中断，等待一段时间后传输失败等问题等问题

        解决方案

        1. 将要上传的大文件用Blob.prototype.slice分成许多大小相同的小块，利用http的可并发性同时上传多个切片。
        2. 所有小块上传完成后，服务器会把这些小块按照正确的顺序拼接起来，还原成完整的大文件。这样就从原本上传一个大文件变成并发多个小文件，减少了上传时间。
        3. 优点：1）可以加快上传速度：可以同时上传多个小块，充分利用网络的带宽。2）减少上传失败的风险：上传过程中出现了问题，只需要重新上传出错的那个小块。

        rbac权限控制的实现

        场景

        1. 不同账号需要展示不同的菜单；比如分析人员可以看到分析相关仪器、试剂，看不到报告管理相关菜单。
        2. 还有一种是是页面中的某些功能，比如是业务员可以申请核销发票，没有审批按钮，而财务有这种功能权限。

        解决方案：RBAC

        1. 传统的权限设计是对每个人进行单独的权限设置，但这种方式已经不适合目前企业的高效管控权限的发展需求。
        2. RBAC（Role-Based Access Control）基于角色的权限分配解决方法，它是在人与权限之前加一个角色的中间层，想对某个用户设置权限，就对该用户设置对应的角色就可。
        3. 就是做角色与权限的映射关系，为角色分配权限。

        组件库之项目实例：使用插件封装多选虚拟长列表

        场景

        1. 系统一些场景的多选选择器，内容过长过多一般有上千条数据，并且选择多个后也不好显示以及删除单个，所以需要一个多选列表组件。
        2. 它有搜索框、可以单选与多选，单选是上下布局，多选显示左右布局。

        解决方案

        1. 使用插件vue-virtual-scroller,它是长列表优化方案之一。
        2. 原理是：它只会把展示给用户的那部分 dom 渲染出来，比如滚到上面的 dom 就会回收掉

        实现

        1. 根据传进来的参数multiple（true\false）决定页面的布局，如果单选则显示为上下布局；多选则为左右布局，右边显示选中的值。
          • 页面的布局根据flex的flex-direction显示。
            • style="display:flex;overflow: auto;" :style="{'flex-direction':mydirection}"
        2. 数据列表使用插件vue-virtual-scroller的RecycleScroller组件展示
          • RecycleScroller组件是做固定高度场景，需指定items itemsize keyfield, keyfieled需唯一稳定
            • 代码：<RecycleScroller style="height: 100%;" :items="renderList" :item-size="32" key-field="name" v-slot="{ item }">
          • 多选模式：主要是每个item都有_checked选项，点击设置item._checked = !item._checked；
          • 搜索关键字：为了用户体验过滤后的列表数据项中的关键字为黄色。
            • 每个数据项会循环匹配是否有keyword，根据replace方法把值替换上带strong标签与样式的关键值。item.text.replace(new RegExp(_firstword, "gm"), <strong class="searchText">${_firstword}</strong>)
        3. 列表自定义选项渲染：比如左边数据列表需要禁选效果，点击可以提示原因,使用作用域插槽实现
          • 列表自定义选项渲染，item值使用插槽占位，方便业务组件传递数据项内容，比如数据列表需要禁选效果，并且点击可以提示原因。
        4. 业务系统使用组件：<fx-listselect style="height: 100%;flex: 1;" layout="row" :listdata="options" v-model="lstCheck"></fx-listselect>

        vue解读源码之前端路由

        SPA是什么

        1. SPA(single page web application),单页Web应用,简单来说就是一个web项目只有一个HTML页面，一旦页面加载完成，就不会因为用户的操作而进行页面的重新加载或跳转，而是利用 JS 动态的变换 HTML 的内容来模拟多个视图间跳转。所以SPA允许在不刷新页面的情况下更改URL并渲染不同内容。
        2. 旧的网站是每个HTML文件都是一个完成的HTML页面，而在SPA 的应用设计中，一个应用只有一个HTML文件。
        3. SPA 的出现大大提高了 WEB 应用的交互体验，但是SPA是通过 JS 改变 HTML 内容来实现的，页面本身的 url 并没有变化，这样会导致刷新、前进还是后退无法被浏览器记录；并且只有一个 url，对 SEO 不友好。
